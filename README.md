# IELTS 口语复习卡片

这是一个面向 IELTS 口语 Part 3 的学生自复习卡片原型。它把题目、答题思路、可背表达、完整答案和中文解释放在同一张卡片里，让学生可以按话题筛选、逐张翻看、先回忆再展开答案。

在线演示：

[打开 IELTS 口语复习卡片](https://yflg2569096112-star.github.io/ielts-flashcard/)

## 项目定位

这个仓库不是完整教学后台，也不是已经接入飞书真实数据的学生端。

它更适合作为项目 B 的一个轻量补充：当项目 B 把老师课堂笔记、课后总结或后续飞书妙记转写内容整理成学生复习卡之后，学生可以用类似这种卡片式界面进行自我复习。

一句话：

> 项目 B 负责把课堂内容沉淀成复习材料；这个页面展示学生拿到材料之后，可以如何自己复习。

## 当前功能

- 按 IELTS 口语 Part 3 话题筛选复习卡。
- 每张卡片先展示题目，帮助学生先自己组织回答。
- 第一层展开显示关键思路和可背表达。
- 第二层展开显示完整英文答案和中文解释。
- 支持上一张、下一张切换。
- 使用静态网页实现，不需要登录，也不连接真实学生数据。

## 适用场景

- 学生课后复习自己的口语题库。
- 老师把课堂输出整理成可翻看的复习卡。
- 面试或作品集展示“学生端复习出口”的轻量形态。
- 作为项目 B 后续学生复习页的交互参考。

## 与项目 B 的关系

项目 B 的主链路是：

```text
老师课堂内容
-> AI 结构化整理
-> 老师确认和质检
-> 飞书多维表格沉淀
-> 学生复习页 / 助教检查视图
```

本仓库只对应其中的“学生复习页 / 学生自复习出口”这一小段。

它可以帮助说明：复习卡不应该只是一次性文本，而应该变成学生能反复打开、按话题查看、逐步回忆和复习的轻量工具。

## 当前边界

- 不写成已接入飞书真实数据。
- 不写成已支持真实学生后台个性化。
- 不写成正式上线或全团队使用。
- 不写成已经验证提分、长期留存或长期打开率。
- 当前内容是 IELTS 口语 Part 3 示例卡片，主要用于展示卡片式复习形态。

## 后续可以扩展

- 把卡片数据从页面内置内容改为外部表格或复习库导入。
- 按学生、课程、话题、错题类型生成不同卡组。
- 增加“已掌握 / 还不熟 / 需要老师讲解”的简单复习状态。
- 与项目 B 的复习卡生成结果衔接，形成更完整的学生端复习体验。
